### 3D 地質模型資料格式規範

根據目前的系統架構與設計規格，3D 地質場景由以下三種資料類型組成。請依據下列規範準備資料：

#### 1. 整體地質/岩體模型 (Major Geological Structures)

用於展示大範圍的地層實體或複雜岩體。

* **格式標準**: **OGC 3D Tiles**
* **檔案結構**:
  * `tileset.json` (索引檔)
  * `.b3dm` (二進位模型檔，內含 GLTF)
* **優勢**: 支援多層次細節 (LOD) 與串流載入，適合網頁端展示巨量模型。
* **對應元件**: `GeologyTiles.tsx`

#### 2. 斷層與構造面 (Fault Planes)

用於展示具方向性的斷層面。此部分採用「參數化」方式，由前端即時運算生成，而非載入靜態模型。

* **格式**: **JSON** (API 回傳)
* **必要參數**:
  * `coordinates`: 地表跡線座標 `[{x, y, z}, ...]` (TWD97)
  * `dipAngle`: 傾角 (0-90度)
  * `dipDirection`: 傾向 (0-360度)
  * `depth`: 延伸深度 (m)
* **對應元件**: `StructureLines.tsx`

#### 3. 鑽孔柱狀圖 (Boreholes)

* **格式**: **JSON**
* **結構**: 需包含孔口座標 (`x,y,z`)、孔深 (`totalDepth`) 及分層資訊 (`layers: [{top, bottom, lithology}, ...]`)。

#### ⚠️ 座標系統要求

所有空間資料必須統一使用 **TWD97 (EPSG:3826)** 平面坐標系 (單位：公尺)，以確保 3D 測量與疊合的精確度。

# 3D 地質模型資料格式規範

根據系統設計規格 (specs/1-geology-module/data-model.md) 與前端實作，3D 地質場景的資料需求分為三大類。以下是各類模型所需的標準格式：

## 1. 整體地質模型 (Volume / Mesh Models)

用於展示大範圍的地層實體、岩體或既有模型 (如從 GOCAD、Leapfrog 匯出者)。

採用標準: OGC 3D Tiles (開放地理空間協會標準)
檔案組成:
tileset.json: 定義模型的空間階層結構與幾何誤差 (LOD 控制)。
.b3dm (Batched 3D Model): 實際的模型檔，封裝了 GLTF 幾何網格與屬性表。
載入方式: 前端透過 GeologyTiles 元件讀取 tileset.json 的 URL 進行串流載入。

## 2. 參數化斷層模型 (Parametric Fault Planes)

用於展示具備地質意義的斷層面。此類模型由前端根據參數「即時運算」生成 3D 幾何。

資料格式: JSON (經由 API 傳輸)
必要欄位:
coordinates: 地表跡線座標陣列 [{x, y, z}, ...] (TWD97)
dipAngle: 傾角 (0-90 度)
dipDirection: 傾向 (0-360 度)
depth: 斷層延伸深度 (公尺)
實作原理: 前端讀取地表跡線後，結合傾角與深度數據，自動向下擠出 (Extrude) 產生 3D 平面。

## 3. 鑽孔與地層資料 (Boreholes)

用於展示離散點位的柱狀圖。

資料格式: JSON
必要結構: 需包含 coordinates (孔口座標)、depth (總深度) 以及 layers (分層資訊與岩性代碼)。
⚠️ 座標系統規範
標準: TWD97 (EPSG:3826) 平面坐標系。
單位: 公尺 (Meter)。
說明: 若原始資料為 WGS84 經緯度，建議在後端 API 層統一轉換為 TWD97 座標後再傳送給前端，以確保 3D 場景中的距離量測精確度。
若您手邊有 .dxf 或 .obj 等傳統 3D 格式，需透過轉檔工具 (如 FME) 轉換為 3D Tiles 格式，才能在本平台上獲得最佳效能。

# 3D 地質模型資料格式規範

根據系統設計規格與前端實作，本平台的 3D 地質場景主要由以下三種資料構成。請依據下列規範準備資料：

## 1. 整體地質模型 (主要地層/岩體)

用於展示大範圍的地層實體模型。

格式標準: OGC 3D Tiles
檔案結構:
入口檔: tileset.json (定義 LOD 階層)
模型檔: .b3dm (包含 GLTF 網格與屬性)
說明: 這是 WebGL 展示巨量 3D 模型的標準格式，支援串流載入。若有 .obj 或 .dxf 檔，需先轉檔為 3D Tiles。

## 2. 斷層與構造平面 (Fault Planes)

這部分採用「參數化」方式，由前端即時運算生成，不需要 3D 模型檔。

格式: JSON (API 回傳)
必要參數:
coordinates: 地表跡線座標 [{x, y, z}, ...] (TWD97)
dipAngle: 傾角 (0-90度)
dipDirection: 傾向 (0-360度)
depth: 延伸深度 (m)

## 3. 鑽孔柱狀圖 (Boreholes)

格式: JSON
結構: 包含孔口座標 (x,y,z)、孔深 (totalDepth) 及分層資訊 (layers 陣列)。
⚠️ 座標系統要求
所有空間資料必須統一使用 TWD97 (EPSG:3826) 平面坐標系 (單位：公尺)，以確保 3D 測量與疊合的精確度。若是經緯度 (WGS84)，請先轉換為 TWD97。
