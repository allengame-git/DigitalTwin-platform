# 3D 地質模型資料格式規範

根據系統設計規格與前端實作，本平台的 3D 地質場景主要由以下三種資料構成。請依據下列規範準備資料：

## 1. 整體地質模型 (主要地層/岩體)

用於展示大範圍的地層實體模型。

格式標準: OGC 3D Tiles
檔案結構:
入口檔: tileset.json (定義 LOD 階層)
模型檔: .b3dm (包含 GLTF 網格與屬性)
說明: 這是 WebGL 展示巨量 3D 模型的標準格式，支援串流載入。若有 .obj 或 .dxf 檔，需先轉檔為 3D Tiles。

## 2. 斷層與構造平面 (Fault Planes)

這部分採用「參數化」方式，由前端即時運算生成，不需要 3D 模型檔。

格式: JSON (API 回傳)
必要參數:
coordinates: 地表跡線座標 [{x, y, z}, ...] (TWD97)
dipAngle: 傾角 (0-90度)
dipDirection: 傾向 (0-360度)
depth: 延伸深度 (m)

## 3. 鑽孔柱狀圖 (Boreholes)

格式: JSON
結構: 包含孔口座標 (x,y,z)、孔深 (totalDepth) 及分層資訊 (layers 陣列)。
⚠️ 座標系統要求
所有空間資料必須統一使用 TWD97 (EPSG:3826) 平面坐標系 (單位：公尺)，以確保 3D 測量與疊合的精確度。若是經緯度 (WGS84)，請先轉換為 TWD97。

相關程式碼參考：

3D 模型載入: src/components/scene/GeologyTiles.tsx
斷層運算: src/components/scene/StructureLines.tsx

## 4. 鑽孔資料 CSV 匯入格式規範

為支援批次匯入鑽孔資料，目前系統支援以下三種 CSV 格式。所有檔案需使用 **UTF-8** 編碼。

## 4.1 鑽孔基本資料 (Borehole Basic Data)

定義鑽孔的位置、高程與基本屬性。

**檔名建議**: `boreholes_basic.csv`
**必要欄位**: `boreholeNo`, `x`, `y`, `elevation`, `totalDepth`

| 欄位名稱 | 說明 | 格式/範例 | 必填 |
| :--- | :--- | :--- | :--- |
| **boreholeNo** | 鑽孔編號 (唯一碼) | BH-001 | 是 |
| **x** | TWD97 東氣坐標 | 224500.0 | 是 |
| **y** | TWD97 北氣坐標 | 2429500.0 | 是 |
| **elevation** | 孔口高程 (m) | 120.5 | 是 |
| **totalDepth** | 總深度 (m) | 50.0 | 是 |
| drilledDate | 鑽探日期 | 2024-01-15 (YYYY-MM-DD) | 否 |
| area | 所屬區域 | A區 | 否 |
| contractor | 鑽探單位 | 工程公司名稱 | 否 |
| description | 備註 | 補充說明 | 否 |

**CSV 範例**:

```csv
boreholeNo,x,y,elevation,totalDepth,drilledDate,area
BH-001,224500,2429500,120.5,50.0,2024-01-15,A區
BH-002,224600,2429600,118.2,45.0,2024-01-20,A區
```

## 4.2 地層資料 (Geological Layers)

定義鑽孔的地層分層資訊。

**檔名建議**: `boreholes_layers.csv`
**必要欄位**: `boreholeNo`, `topDepth`, `bottomDepth`, `lithologyCode`

| 欄位名稱 | 說明 | 格式/範例 | 必填 |
| :--- | :--- | :--- | :--- |
| **boreholeNo** | 鑽孔編號 | BH-001 (需對應基本資料) | 是 |
| **topDepth** | 頂部深度 (m) | 0.0 | 是 |
| **bottomDepth** | 底部深度 (m) | 2.5 | 是 |
| **lithologyCode** | 岩性代碼 | SF (回填土), SM, CL... | 是 |
| description | 地層描述 | 深褐色黏土夾安山岩塊 | 否 |
| lithologyName | 岩性名稱 | 回填土 | 否 |

**CSV 範例**:

```csv
boreholeNo,topDepth,bottomDepth,lithologyCode,description
BH-001,0,2.5,SF,回填土
BH-001,2.5,8.0,SM,砂質粉土
BH-001,8.0,15.0,CL,低塑性黏土
BH-002,0,3.0,SF,回填土
```

## 4.3 物性資料 (Physical Properties)

定義鑽孔沿深度的物理性質測試數據 (如 SPT-N 值、RQD)。

**檔名建議**: `boreholes_properties.csv`
**必要欄位**: `boreholeNo`, `depth`

| 欄位名稱 | 說明 | 格式/範例 | 必填 |
| :--- | :--- | :--- | :--- |
| **boreholeNo** | 鑽孔編號 | BH-001 | 是 |
| **depth** | 測試深度 (m) | 1.5 | 是 |
| nValue | SPT-N 值 | 12 (整數) | 否 |
| rqd | 岩石品質指標 (%) | 85 (0-100) | 否 |

**CSV 範例**:

```csv
boreholeNo,depth,nValue,rqd
BH-001,1.5,5,
BH-001,3.0,8,
BH-001,4.5,12,
BH-001,10.0,,85
BH-002,2.0,8,
```

## 4.4 岩性定義資料 (Lithology Definitions)

定義專案中使用的岩性代碼、名稱與對應顏色。這對於確保 3D 模型與鑽孔剖面顏色一致非常重要。

**欄位說明**:

| 欄位名稱 | 說明 | 格式/範例 | 必填 |
| :--- | :--- | :--- | :--- |
| **lithId** | 岩性編號 (整數) | 1 (用於 3D Shader 對應，需唯一) | 是 |
| **code** | 岩性代碼 | SF, CL, GM (需與地層資料對應) | 是 |
| **name** | 岩性名稱 | 回填土, 低塑性黏土, 礫石層 | 是 |
| **color** | 顯示顏色 (Hex) | #8b4513 | 是 |

**資料範例**:

| lithId | code | name | color |
| :--- | :--- | :--- | :--- |
| 1 | SF | 回填土 | #eab308 |
| 2 | SM | 砂質粉土 | #22c55e |
| 3 | CL | 低塑性黏土 | #ef4444 |
| 4 | GM | 礫石層 | #3b82f6 |
| 5 | SD | 砂土 | #f59e0b |

## 5. 斷層面 CSV 匯入格式規範 (Fault Plane CSV)

用於批次匯入斷層構造資料。

**檔名建議**: `fault_planes.csv`
**必要欄位**: `name`, `type`, `dipAngle`, `dipDirection`, `depth`, `coordinates`

| 欄位名稱 | 說明 | 格式/範例 | 必填 |
| :--- | :--- | :--- | :--- |
| **name** | 斷層名稱 | 主斷層 A | 是 |
| **type** | 斷層類型 | normal (正), reverse (逆), strike-slip (走滑) | 是 |
| **dipAngle** | 傾角 (度) | 70 (範圍 0-90) | 是 |
| **dipDirection** | 傾向 (度) | 90 (範圍 0-360) | 是 |
| **depth** | 延伸深度 (m) | 200.0 | 是 |
| color | 顯示顏色 (Hex) | #ff4444 | 否 |
| **coordinates** | 點序列 JSON | "[{""x"":250000,""y"":2600000,""z"":0}, ...]" | 是 |

### ⚠️ 特別注意: coordinates 格式

coordinates 欄位必須是一個包含點座標的 **JSON 陣列字串**。在 CSV 中，JSON 內部的雙引號需使用「兩個雙引號 (`""`)」進行轉義。

**CSV 範例**:

```csv
name,type,dipAngle,dipDirection,depth,color,coordinates
主斷層 A,normal,70,90,200,#ff4444,"[{""x"":224500,""y"":2429500,""z"":10},{""x"":224600,""y"":2429600,""z"":0}]"
次要斷層 B,reverse,45,270,150,#4488ff,"[{""x"":224000,""y"":2429000,""z"":5},{""x"":224100,""y"":2429100,""z"":-5}]"
```

## 6. 位態資料 CSV 匯入格式規範 (Attitude Data CSV)

定義地質調查中的位態測量資料（走向與傾角）。

**檔名建議**: `attitudes.csv`
**必要欄位**: `x`, `y`, `z`, `strike`, `dip`

| 欄位名稱 | 說明 | 格式/範例 | 必填 |
| :--- | :--- | :--- | :--- |
| **x** | TWD97 東坐標 | 224500.0 | 是 |
| **y** | TWD97 北坐標 | 2429500.0 | 是 |
| **z** | 高程 (m) | 120.5 | 是 |
| **strike** | 走向 (0-360度) | 45.5 | 是 |
| **dip** | 傾角 (0-90度) | 30.0 | 是 |
| dipDirection | 傾向 (文字) | N, NE, E, SE, S, SW, W, NW | 否 |
| description | 備註 | 節理面, 層面 | 否 |

**CSV 範例**:

```csv
x,y,z,strike,dip,dipDirection,description
224500,2429500,120.5,45,30,NE,節理面-1
224600,2429600,118.2,120,45,SE,層面-2
```
