# Feature Specification: 模擬分析展示模組 (Simulation & Analysis Module)

**Status**: DRAFT  
**Owner**: Engineering Team  
**Created**: 2026-02-02

## 1. Overview

本模組提供工程數位孿生平台的數值模擬結果視覺化功能。使用者可透過體積渲染、等值面與熱圖等方式檢視地下水污染傳輸、流場分佈等模擬成果，並透過情境切換比較不同模擬條件的結果。

**核心價值**:

- 將複雜的數值模擬結果轉化為直觀的 3D 視覺化
- 支援多情境比較，輔助工程決策
- 整合儀表板與 3D 場景的雙向互動，提升分析效率

## 2. User Scenarios

### 情境 A：工程師分析污染擴散

作為**工程師**，我想要**在 3D 場景中檢視污染羽的空間分佈與時間演化**，以便**評估污染範圍與整治成效**。

**流程**：

1. 開啟模擬分析模組，載入污染傳輸模擬結果
2. 選擇體積渲染或等值面顯示模式
3. 播放時間序列動畫觀察污染擴散過程
4. 點擊特定位置查看該點的濃度時間歷線

### 情境 B：工程師比較模擬情境

作為**工程師**，我想要**切換不同模擬情境並比較結果**，以便**選擇最佳工程方案**。

**流程**：

1. 使用下拉選單選擇「豐水期」情境
2. 檢視 3D 場景與儀表板數據
3. 切換至「枯水期」情境
4. 比較兩情境的關鍵指標差異

### 情境 C：審查委員檢核模擬設定

作為**審查委員**，我想要**檢視模擬的輸入參數與輸出統計**，以便**評估模擬結果的可信度**。

**流程**：

1. 開啟儀表板面板
2. 檢視模擬輸入參數表格
3. 對照輸出結果統計圖表
4. 點擊特定數據點連動至 3D 場景定位

### 情境 D：民眾了解環境影響

作為**一般民眾**，我想要**透過簡化的視覺化了解工程對地下水的影響**，以便**理解環境保護措施**。

**流程**：

1. 進入導覽模式
2. 觀看比較動畫：有/無整治措施的差異
3. 閱讀關鍵指標的簡易說明

## 3. Functional Requirements

### 數值模擬視覺化

- [ ] FR-01: 系統必須支援體積渲染 (Volumetric Rendering) 顯示 3D 濃度場
- [ ] FR-02: 系統必須支援等值面 (Isosurface) 提取，使用者可調整閾值
- [ ] FR-03: 系統必須支援在地表或剖面繪製濃度/壓力熱圖 (Heatmap)
- [ ] FR-04: 系統必須支援流線圖 (Streamlines) 顯示地下水流場

### 時序動畫

- [ ] FR-05: 系統必須支援時間序列模擬結果的動畫播放
- [ ] FR-06: 時間軸必須顯示當前模擬時間步與對應日期
- [ ] FR-07: 使用者必須能暫停並逐步檢視各時間步結果

### 情境分析

- [ ] FR-08: 系統必須提供下拉選單切換預定義的模擬情境
- [ ] FR-09: 切換情境時，3D 場景與儀表板必須同步更新
- [ ] FR-10: 系統必須支援至少 20 種預定義情境

### 儀表板

- [ ] FR-11: 儀表板必須以表格顯示模擬輸入參數
- [ ] FR-12: 儀表板必須以 ECharts 圖表顯示輸出結果統計
- [ ] FR-13: 點擊 3D 場景位置必須連動更新 ECharts 時間歷線圖表
- [ ] FR-14: 點擊 ECharts 圖表數據點必須連動 3D 場景高亮對應位置

### 資料處理

- [ ] FR-15: 系統必須支援載入抽稀後的網格數據 (簡化 VTK/NetCDF 格式)
- [ ] FR-16: 時間歷線資料必須支援 JSON 格式載入
- [ ] FR-17: 導覽模式內容 (動畫路徑、說明文字、停留點) 必須透過外部 JSON 配置檔載入

## 4. Non-Functional Requirements

### 效能

- [ ] NFR-01: 體積渲染必須維持 30 FPS 以上的互動流暢度
- [ ] NFR-02: 情境切換必須在 2 秒內完成場景更新
- [ ] NFR-03: 時間序列動畫播放不得有明顯掉幀

### 相容性

- [ ] NFR-04: 大型模擬資料必須支援分級載入 (依相機距離調整精細度)
- [ ] NFR-05: 系統必須支援 WebGL 2.0 相容的設備

### 可用性

- [ ] NFR-06: 熱圖/等值面的色彩圖例 (Color Legend) 必須清楚標示數值範圍
- [ ] NFR-07: 情境切換時必須顯示載入指示器

### 異常處理

- [ ] NFR-08: 模擬資料載入失敗時必須顯示友善錯誤訊息與重試按鈕
- [ ] NFR-09: 體積渲染無法執行時必須提供降級顯示 (2D 熱圖替代)

## 5. Success Criteria

- [ ] SC-01: 工程師能在 1 分鐘內完成兩情境的視覺比較
- [ ] SC-02: 3D 場景與 ECharts 圖表的雙向連動延遲 <500ms
- [ ] SC-03: 使用者能正確識別污染羽超標區域 (等值面閾值設定後)
- [ ] SC-04: 90% 的民眾表示導覽模式「易於理解」

## 6. Assumptions & Constraints

### Assumptions

- 模擬結果由專業軟體產出，已轉為 Web 友善格式
- 模擬情境的定義 (參數組合) 由專案團隊預先設定
- 使用者設備的 GPU 支援體積渲染運算

### Constraints

- 首期不支援使用者自訂模擬參數與重新運算
- 原始高解析度模擬資料需在伺服器端預處理 (抽稀/轉檔)
- 即時模擬運算不在範圍內

## 7. Data Model

### 核心實體

```
SimulationScenario (模擬情境)
├── id: string
├── name: string (情境名稱, e.g., "豐水期-有整治")
├── description: string
├── inputParameters: { [key: string]: any }
├── volumeDataUrl: string (體積資料 URL)
├── timeSteps: TimeStep[]
└── createdAt: Date

TimeStep (時間步)
├── id: string
├── scenarioId: string
├── stepIndex: number
├── simulationTime: number (模擬時間)
├── date: Date (對應實際日期)
└── dataUrl: string (該時間步資料)

ObservationPoint (觀測點)
├── id: string
├── name: string
├── coordinates: { x, y, depth } (TWD97 EPSG:3826)
├── timeSeries: { time: Date, value: number }[]
└── unit: string (單位)
```
