# Feature Specification: 使用者角色與權限系統 (User Roles & Permissions System)

**Status**: DRAFT  
**Owner**: Engineering Team  
**Created**: 2026-02-02

## 1. Overview

本模組定義工程數位孿生平台的使用者角色與存取權限機制。系統區分三種核心角色：工程師、審查委員與一般民眾，各有不同的功能存取範圍與介面呈現方式。

**核心價值**:

- 確保專業資料僅對授權人員開放
- 為不同使用者提供最適化的介面體驗
- 簡化民眾視角，避免技術細節造成困惑

## 2. User Scenarios

### 情境 A：工程師完整存取

作為**工程師**，我想要**存取所有功能與原始數據**，以便**進行深度技術分析與驗證**。

**流程**：

1. 使用帳號密碼登入系統
2. 進入完整功能介面
3. 可切換任意圖層、存取原始數據、使用量測工具
4. 可匯出資料供進一步分析

### 情境 B：審查委員檢視與標註

作為**審查委員**，我想要**檢視關鍵設計資料並記錄審查意見**，以便**完成設計審查工作**。

**流程**：

1. 使用受邀連結或帳號登入
2. 進入審查模式介面
3. 檢視工程模型與模擬結果
4. 使用標註工具記錄意見
5. 提交審查報告

### 情境 C：民眾瀏覽導覽

作為**一般民眾**，我想要**無需登入即可瀏覽科普內容**，以便**了解工程概況**。

**流程**：

1. 透過公開連結進入平台
2. 自動進入導覽模式
3. 觀看預設動畫與說明
4. 技術參數與複雜介面元素已隱藏

## 3. Functional Requirements

### 角色定義

- [ ] FR-01: 系統必須支援「工程師 (Engineer)」角色，擁有完整功能存取權限
- [ ] FR-02: 系統必須支援「審查委員 (Reviewer)」角色，擁有檢視與標註權限
- [ ] FR-03: 系統必須支援「一般民眾 (Public)」角色，僅能存取導覽模式

### 工程師權限

- [ ] FR-04: 工程師必須能存取所有圖層與資料
- [ ] FR-05: 工程師必須能切換複雜圖層組合
- [ ] FR-06: 工程師必須能使用量測工具 (距離、面積、角度)
- [ ] FR-07: 工程師必須能查看原始數據檔案
- [ ] FR-08: 工程師必須能匯出視圖截圖與數據報表

### 審查委員權限

- [ ] FR-09: 審查委員必須能檢視所有預設展示內容
- [ ] FR-10: 審查委員必須能使用標註工具 (文字、箭頭、區域框選)
- [ ] FR-11: 標註必須能儲存並由其他審查委員查看
- [ ] FR-12: 審查委員必須能切換設計驗證相關圖層
- [ ] FR-13: 複雜技術操作 (如原始資料存取) 對審查委員隱藏

### 民眾權限

- [ ] FR-14: 民眾必須能無需登入即可存取導覽模式
- [ ] FR-15: 導覽模式必須隱藏技術參數與專業術語
- [ ] FR-16: 導覽模式必須以 Storytelling 方式呈現，自動播放 3D 動畫
- [ ] FR-17: 導覽模式必須提供「了解更多」按鈕連結至圖文說明
- [ ] FR-18: 民眾模式的介面必須極度簡化，僅保留播放控制與探索按鈕

### 認證與授權

- [ ] FR-19: 工程師與審查委員必須透過帳號密碼登入
- [ ] FR-20: 系統必須支援審查委員的邀請連結登入方式
- [ ] FR-21: 登入 Session 逾時機制：工程師 8 小時無活動自動登出，審查委員 1 小時無活動自動登出

### 介面適配

- [ ] FR-22: 系統必須根據角色動態調整介面佈局
- [ ] FR-23: 工程師介面必須顯示完整工具列與圖層控制面板
- [ ] FR-24: 審查委員介面必須突出標註工具與審查相關功能
- [ ] FR-25: 民眾介面必須僅顯示導覽控制按鈕

## 4. Non-Functional Requirements

### 安全性

- [ ] NFR-01: 敏感資料 (原始數據、設計圖說) 必須僅對授權角色開放
- [ ] NFR-02: API 端點必須驗證使用者角色權限
- [ ] NFR-03: 密碼必須加密儲存

### 效能

- [ ] NFR-04: 角色權限檢查必須在每次頁面載入時完成，延遲 <100ms
- [ ] NFR-05: 介面適配切換必須在登入完成後立即生效

### 可用性

- [ ] NFR-06: 使用者必須清楚知道自己目前的角色與可用功能
- [ ] NFR-07: 當使用者嘗試存取無權限功能時，必須顯示友善提示

### 可觀測性

- [ ] NFR-08: 系統必須記錄前端錯誤日誌並回傳至伺服器
- [ ] NFR-09: 系統必須收集基本頁面瀏覽統計 (頁面訪問次數、停留時間)

## 5. Success Criteria

- [ ] SC-01: 工程師能在登入後 5 秒內存取完整功能
- [ ] SC-02: 審查委員的標註功能使用率達 80% (有標註的審查會議佔比)
- [ ] SC-03: 民眾導覽模式的跳出率 <30%
- [ ] SC-04: 無未授權存取安全事件發生

## 6. Assumptions & Constraints

### Assumptions

- 使用者帳號由系統管理員預先建立
- 審查委員會議期間有穩定網路連線
- 民眾使用者不需註冊或登入

### Constraints

- 首期不支援自助註冊
- 角色權限為固定組合，不支援自訂權限細項
- 標註資料儲存於伺服器，不支援離線標註

## 7. Data Model

### 核心實體

```
User (使用者)
├── id: string
├── email: string
├── passwordHash: string
├── name: string
├── role: 'engineer' | 'reviewer' | 'public'
├── createdAt: Date
└── lastLoginAt: Date

Annotation (標註)
├── id: string
├── userId: string (標註者)
├── projectId: string
├── type: 'text' | 'arrow' | 'region'
├── content: string
├── position: { x, y, z } (3D 座標)
├── cameraState: object (當時視角)
├── createdAt: Date
└── isResolved: boolean

InviteLink (邀請連結)
├── id: string
├── token: string (唯一 Token)
├── targetRole: 'reviewer'
├── expiresAt: Date
├── usedBy: string | null
└── createdAt: Date
```
