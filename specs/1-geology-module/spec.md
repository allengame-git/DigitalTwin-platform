# Feature Specification: 地質資料展示模組 (Geology & Hydrogeology Module)

**Status**: DRAFT  
**Owner**: Engineering Team  
**Created**: 2026-02-02

## 1. Overview

本模組提供工程數位孿生平台的地質與水文資料視覺化功能。使用者可透過 3D 互動介面檢視地質模型、鑽探資料與地表構造，滿足專業工程師的深度分析需求，同時也能產出民眾易懂的科普展示。

**核心價值**:

- 整合分散的地質資料（鑽探、地球物理、地表構造）於單一 3D 場景
- 透過 LOD 機制處理數百個鑽孔資料點，確保效能與資訊豐富度兼顧
- 支援專業人員的細部檢視與一般民眾的導覽模式

## 2. User Scenarios

### 情境 A：工程師檢視地質剖面

作為**工程師**，我想要**透過切片工具檢視 3D 地質模型的內部結構**，以便**評估地層分布與潛在地質風險**。

**流程**：

1. 開啟地質展示模組，載入專案區域的 3D 地質模型
2. 使用切片工具設定剖面位置與角度
3. 調整透明度以同時檢視表層與深層地質
4. 點擊特定地層取得詳細物性資料

### 情境 B：工程師分析鑽探資料

作為**工程師**，我想要**在 GIS 地圖上快速定位特定鑽孔並檢視岩芯資料**，以便**驗證地質模型的準確性**。

**流程**：

1. 在地圖視圖中瀏覽鑽孔點位分布
2. 縮放至感興趣區域，點位自動顯示岩性柱狀圖
3. 點擊特定孔位開啟詳細面板
4. 檢視岩芯照片、物性數據與地下水位

### 情境 C：審查委員比對設計與地質

作為**審查委員**，我想要**疊加航照圖與地質構造線於 3D 場景**，以便**檢核工程設計是否考量斷層位置**。

**流程**：

1. 開啟圖層控制面板
2. 啟用航照圖、DEM 地形與斷層線圖層
3. 旋轉視角檢視構造線與工程設施的關係
4. 使用標註工具記錄審查意見

### 情境 D：民眾瀏覽科普導覽

作為**一般民眾**，我想要**透過導覽模式了解工區的地質特性**，以便**理解工程設計的背景**。

**流程**：

1. 進入導覽模式，系統自動播放預設動畫
2. 觀看簡化的 3D 地質結構與說明文字
3. 點擊「了解更多」檢視圖文解說
4. 可暫停導覽並自由探索

## 3. Functional Requirements

### 3D 地質模型

- [ ] FR-01: 系統必須載入並渲染 3D 地質模型，支援至少 5 個獨立地層的顯示
- [ ] FR-02: 使用者必須能透過切片工具在任意角度切開模型，即時顯示剖面
- [ ] FR-03: 使用者必須能調整各地層的透明度 (0%-100%)，支援同時檢視多個深度
- [ ] FR-04: 系統必須支援爆炸圖模式，將地層垂直分離以便檢視

### 鑽探資料庫

- [ ] FR-05: 系統必須在 GIS 地圖上顯示所有鑽孔點位 (支援至少 800 個孔位)
- [ ] FR-06: 系統必須實作 LOD 機制：視角距離 >2km 顯示點標記，<2km 顯示岩性柱狀圖
- [ ] FR-07: 點擊孔位時，系統必須彈出詳細面板顯示岩芯照片、物性數據與地下水位
- [ ] FR-08: 詳細面板必須與 ECharts 圖表連動，顯示深度-物性數據曲線

### 地表與構造

- [ ] FR-09: 系統必須支援疊加高解析度航照圖 (最高支援 0.5m 解析度)
- [ ] FR-10: 系統必須支援載入 DEM 地形資料並以 3D 地形呈現
- [ ] FR-11: 系統必須支援顯示地質構造線 (斷層)，使用線段符號標示
- [ ] FR-12: 系統必須支援顯示地質位態符號 (Strike/Dip)，以 3D 圓盤或法向量呈現

### 資料處理 (Phase 2 範圍)

> 📌 以下功能規劃於第二階段實作，首期暫不涵蓋

- [ ] FR-13: 系統必須支援匯入鑽探資料 CSV/Excel 格式，包含孔口座標與地層資訊
- [ ] FR-14: 系統必須支援載入電阻率/電磁法反演剖面，以帶座標的垂直貼圖呈現
- [ ] FR-15: 系統必須支援載入震測資料，以點雲或網格方式呈現地層界面

### 視覺控制

- [ ] FR-16: 使用者必須能透過圖層控制面板開關所有地質圖層
- [ ] FR-17: 系統必須提供地下透明度滑桿，統一控制地表以下物件的可見度
- [ ] FR-18: 導覽模式內容 (動畫路徑、說明文字、停留點) 必須透過外部 JSON 配置檔載入

## 4. Non-Functional Requirements

### 效能

- [ ] NFR-01: 載入 800 個鑽孔點位時，初始渲染必須在 3 秒內完成
- [ ] NFR-02: 切片工具操作時，畫面更新延遲必須小於 200ms
- [ ] NFR-03: 透明度調整必須即時反映，無明顯卡頓

### 相容性

- [ ] NFR-04: 系統必須支援主流瀏覽器 (Chrome, Firefox, Safari, Edge 最新兩個版本)
- [ ] NFR-05: 系統必須支援觸控設備的基本操作 (平移、縮放、旋轉)

### 可用性

- [ ] NFR-06: 載入大型資產時必須顯示進度指示器
- [ ] NFR-07: 所有互動元素必須有 Tooltip 提示說明

### 異常處理

- [ ] NFR-08: 3D 資產載入失敗時必須顯示友善錯誤訊息與重試按鈕
- [ ] NFR-09: 3D 模型無法載入時必須提供降級顯示 (2D 地圖替代)

## 5. Success Criteria

- [ ] SC-01: 工程師能在 30 秒內找到特定鑽孔並檢視其岩芯資料
- [ ] SC-02: 系統能同時顯示 800 個鑽孔且維持流暢互動 (>30 FPS)
- [ ] SC-03: 使用者滿意度調查中「地質資料清晰度」評分達 4/5 以上
- [ ] SC-04: 導覽模式完成率 (播放至結束) 達 70% 以上

## 6. Assumptions & Constraints

### Assumptions

- 鑽探資料將以 CSV/Excel 格式提供，包含標準化的欄位名稱
- 地質模型已預先建立並以標準 3D 格式 (glTF/3D Tiles) 提供
- 使用者設備具備支援 WebGL 2.0 的顯示卡

### Constraints

- 首期僅支援單一專案區域，不支援多專案切換
- 離線模式不在首期範圍內
- 岩芯照片需預先處理為 Web 友善格式 (JPEG, 最大 2MB/張)

## 7. Data Model

### 核心實體

```
Borehole (鑽孔)
├── id: string (唯一識別碼)
├── name: string (孔位名稱)
├── coordinates: { x, y, z } (孔口座標，WGS84)
├── depth: number (總深度，公尺)
├── waterLevel: number (地下水位深度)
├── layers: Layer[] (地層陣列)
└── photos: Photo[] (岩芯照片陣列)

Layer (地層)
├── id: string
├── boreholeId: string (所屬鑽孔)
├── topDepth: number (頂深)
├── bottomDepth: number (底深)
├── lithology: string (岩性描述)
└── properties: { [key: string]: number } (物性數據)

GeologicalStructure (地質構造)
├── id: string
├── type: 'fault' | 'fold' | 'strike_dip'
├── geometry: GeoJSON
└── attributes: { [key: string]: any }
```
